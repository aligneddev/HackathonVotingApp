## Implementation Details

### Service Layer: `LeaderboardService`

**File:** `src/HackathonVotingApp.Api/Services/LeaderboardService.cs`

**Implementation:**
```csharp
public async Task<List<LeaderboardEntryResponse>> GetLeaderboardAsync(int limit = 50)
{
    var presentations = await _context.Presentations.ToListAsync();
    
    var entries = presentations
        .Join(_votes, p => p.Id, v => v.PresentationId,
              (p, v) => new { p.Id, p.Title, p.PresenterName, v.Id })
        .GroupBy(x => new { x.Id, x.Title, x.PresenterName })
        .OrderByDescending(g => g.Count())
        .Take(limit)
        .Select(g => new LeaderboardEntryResponse(
            g.Key.Id, g.Key.Title, g.Key.PresenterName, g.Count()))
        .ToList();
    
    return entries;
}
```

**Key decisions:**
- In-memory aggregation using LINQ-to-Objects (not LINQ-to-SQL) for flexibility
- Default limit changed to **50** per Kevin's approval in red phase
- Real-time: queries database each call (no caching — fresh vote counts every request)
- Presentation without votes: filtered out by join (only ranked presentations shown)

```csharp
leaderboard.MapGet("/{limit?}", async (int? limit, ILeaderboardService service) =>
{
    var result = await service.GetLeaderboardAsync(limit ?? 50);
    return Results.Ok(result);
})
.WithName("GetLeaderboard")
.WithOpenApi();
```

**Query parameter:** `limit` (optional, default 50)

**Response:** 200 OK with array of `LeaderboardEntryResponse` objects, ranked highest-vote-first.
